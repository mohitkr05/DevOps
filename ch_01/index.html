Building Software

Let us how software is built. Traditionally, the procedure to build software was slow and linear. Usually termed as a waterfall model had each step dedidcated to a single task.

Waterfall model

Each step tried to answer some simple question.

  1. Requirements
  2. What are the requirements of the customer?
  3. Design
  4. How to build the software with all these requirements?
  5. How the high level design looks like?
  6. What are the components required
  7. Implementation
  8. The actual building of the software?
  9. Verfication
  10. How is the software performing as per the requirements of the customer?
  11. Maintenance
  12. Are there any operational issues, bug-fixes and patches required?

Advantages

  • Simple to implement
  • Clear requirements

Problem with this methodology

  • Slow
  • Resistant to change
  • Feedback?
  • Scoping issues
  • Integration is painful.
